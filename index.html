Raspberry Pi Pico Environmental Test Kit

Raspberry Pi PicoFreeRTOSI2C / SPI / ADC / PWM

Helped create a Raspberry Pi Pico test kit that acts as a hub for environmental sensor readings. The system used a temperature and humidity sensor, pressure sensor, photoresistor light sensor, 240x240 LCD screen, and a custom D-pad made from buttons. The user could switch between different sensor screens and view real-time data through visual displays. The project used I2C for the pressure and temperature/humidity sensors, SPI for the display, ADC for the light sensor, GPIO for buttons, and PWM for display brightness control. The system was organized with FreeRTOS tasks so the buttons, sensors, display, and stopwatch could run separately.

  • Integrated temperature, humidity, pressure, and light sensors
  • Built a user interface with an LCD screen and custom D-pad
  • Used FreeRTOS tasks to organize sensor, button, display, and stopwatch functions
  • Used an I2C mutex to prevent communication conflicts between shared sensors
  • Displayed sensor data through visual UI screens, including gauges, bars, and changing graphics

MSP430 Guitar Hero Game

MSP430F5529Timers / InterruptsPWM

Helped build a Guitar Hero-style game on the MSP430F5529 LaunchPad. The game played a song using a buzzer, flashed LEDs to match notes, and required the user to press the correct button during a timing window. The project used digital I/O, timers, interrupts, PWM sound generation, and LCD display integration. A Timer A2 interrupt updated a global timer every 5 ms, which allowed the game to track note durations and check user input without blocking the CPU.

  • Programmed real-time note timing using Timer A2 interrupts
  • Used PWM to generate buzzer tones for different musical notes
  • Mapped song notes to LED outputs and button inputs
  • Created data structures to store pitch, duration, and LED values
  • Added scoring, mistake tracking, win/loss screens, and LCD feedback

MSP430 Simon Game

MSP430F5529Finite State MachineLCD / Keypad

Helped implement a Simon-style memory game using the MSP430F5529 board. The game flashed a growing pseudo-random LED sequence and required the player to repeat it using corresponding buttons. The system used a finite state machine with states for the welcome screen, countdown, sequence playback, user input, checking input, and updating the game. The game also used the LCD screen for instructions and feedback, LEDs for visual cues, a buzzer for sound, and keypad controls for start, stop, and reset functions.

  • Designed game flow using a finite state machine
  • Integrated LEDs, buttons, keypad input, buzzer sounds, and LCD feedback
  • Stored and compared user input against a generated sequence
  • Added increasing difficulty by reducing the delay as rounds progressed
  • Implemented reset and stop controls for better usability
